# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working
with code in this repository.

## What this is

`vchat-web` is a React 19 SPA that renders YouTube live-chat archives
produced by the upstream **honeybee** archiver. It replaces honeybee's
previous static-HTML output. The SPA loads `index.json`, per-channel
`channel.json`, per-video `videoMeta.json`, and per-video `rows.jsonl`
from the same origin under `/data/…` and renders three views: index
list, channel page, video page.

Full design and implementation history live in
[docs/superpowers/specs/](docs/superpowers/specs/) and
[docs/superpowers/plans/](docs/superpowers/plans/). Read those before
making non-trivial changes.

## Commands

Requires Node ≥ 24 (see `.nvmrc`).

```bash
npm install
npm run dev          # Vite dev server on http://localhost:5173
npm run typecheck    # tsr generate + tsc --noEmit
npm run lint         # eslint .
npm run build        # tsr generate + tsc --noEmit + vite build
npm run preview      # serve dist/
```

`typecheck` and `build` regenerate `src/routeTree.gen.ts` first. Do
not edit that file by hand.

For local dev against a real data source, copy `.env.example` to
`.env.local` and set `VITE_DATA_PROXY` to a deployment that serves the
honeybee `/data` tree. If unset, `/data/*` requests will 404 (proxy
disabled with a warning — see `vite.config.ts`).

## Architecture

### Data ownership boundary

The SPA never writes data. **Honeybee owns the `/data/` prefix on S3**;
the SPA owns everything else at the bucket root. The deploy workflow's
`aws s3 sync --delete --exclude "data/*"` enforces this. New SPA
top-level prefixes are fine; new honeybee prefixes require extending
the exclude list in `.github/workflows/deploy.yml`. See
[README.md](README.md) §"S3 layout contract".

### Routing (TanStack Router, file-based)

Routes live in [src/routes/](src/routes/). File names map to URL
segments:

- `__root.tsx` — layout shell with `TopBar` + `<Outlet>`
- `index.tsx` — `/` with `?tab=live|past`
- `channels.$channelId.tsx` — `/channels/:channelId`
- `videos.$videoId.tsx` — `/videos/:videoId` with
  `?types=[…]&sigRange=[lo,hi]`

`src/routeTree.gen.ts` is auto-generated by `tsr generate` (the
`@tanstack/router-plugin/vite` plugin in `vite.config.ts`).
`src/router.ts` wires the generated tree, JSON search-param
serializer, and `defaultPreload: "intent"`.

Search params are validated with zod schemas per route. URL is the
source of truth for filter state — components call
`navigate({ search: { ... }, replace: true })`.

### MUI + TanStack Router polymorphic typing

MUI v6's `component="Link"` polymorphic typing rejects TanStack
Router's `params`/`search` props. Use the wrappers in
[src/lib/createMuiLink.ts](src/lib/createMuiLink.ts) (`RouterBox`,
`RouterCardActionArea`, `RouterChip`) instead of trying to compose
MUI + Router inline.

### Data layer

- [src/api/types.ts](src/api/types.ts) — `ChatRow` discriminated union
  plus all server payload shapes. Single source of truth.
- [src/api/client.ts](src/api/client.ts) — `fetchJson`, `fetchJsonl`,
  `NotFoundError`.
- [src/api/queries.ts](src/api/queries.ts) — TanStack Query hooks. All
  accept `string | undefined` and gate on `enabled: Boolean(id)`;
  retry skips 404 via `retryOnNon404`.
- [src/lib/jsonl.ts](src/lib/jsonl.ts) — `parseJSONL` with
  zod-validated `type` enum and per-line `try`/`catch`. Unknown or
  malformed rows are skipped via `warnOnce`.

### Chat row rendering

[src/components/chat-rows/](src/components/chat-rows/) holds one
component per `ChatRow.type` plus shared infra:

- `RowDispatcher.tsx` — `switch (row.type)` with `never`
  exhaustiveness; wrapped in `React.memo` (virtualized scroll hot
  path).
- `ChatRowBase.tsx` — shared grid template
  (`56px 180px 56px 200px 1fr`).
- `TimestampLink.tsx` — YouTube deep link with `?t=` offset when the
  row falls after `actualStart`.
- `constants.ts` — `MEMBERSHIP_GREEN`, `MODERATOR_BLUE`,
  `VERIFIED_GREY`.

Adding a new row type: extend the `ChatRow` union in `types.ts`, add
the literal to `rowTypeSchema` in `jsonl.ts`, add a `<Type>Row.tsx`,
and wire it into `RowDispatcher` (TS will error in the `never` branch
until you do).

### Virtualization

[src/components/ChatList.tsx](src/components/ChatList.tsx) uses
`@tanstack/react-virtual`. Heights are dynamic via `measureElement`
and `data-index`. The Box height is
`calc(100vh - var(--topbar-h) - headerHeight - CHATLIST_PAD_PX)` —
`--topbar-h` is set globally by `TopBar` (always mounted),
`headerHeight` is per-video via a `ResizeObserver` in the video
route. Do not reintroduce a global `--chatlist-height` CSS var — it
races with route preload.

The file has an
`eslint-disable-next-line react-hooks/incompatible-library` for
`useVirtualizer`; see the comment for the upstream issue link before
changing.

### Settings

[src/lib/settings.ts](src/lib/settings.ts) exports `useTimezonePref`,
`useThemePref`, `useResolvedTheme`. All built on `useSyncExternalStore`
(correct pattern for `localStorage` + `matchMedia` subscription; do
not rewrite as `useState` + `useEffect`).

### Warnings

[src/lib/warn.ts](src/lib/warn.ts) — `warnOnce(label, detail)`. Used
everywhere unexpected runtime state is encountered (unknown enum
value, malformed JSONL, Slider out-of-shape, etc.). Keys are
truncated at 200 chars to prevent memory bloat. Use this instead of
bare `console.warn`.

## Conventions

- **ESLint** — `src/routes/**` exempts
  `react-refresh/only-export-components` (file-based routes
  legitimately export `Route` alongside local components).
  `src/routeTree.gen.ts`, `dist/`, and `.claude/` are globally
  ignored (the last because agent worktrees there otherwise trigger
  tsconfig collisions).
- **No `verbatimModuleSyntax`** in `tsconfig.app.json` — value
  imports of MUI components do not need `import type`.
- **External links** — always
  `target="_blank" rel="noopener noreferrer"`. YouTube `<img>` use
  `referrerPolicy="no-referrer"`.
- **Status text** — `StatusText.tsx` is the canonical place to render
  `VideoStatus`; route to it from any new surface that shows status.
- **Document title** — `useDocumentTitle` does NOT restore on unmount
  (avoids a route-preload race). Each route must call it explicitly.

## Deployment

Push to `main` triggers `.github/workflows/deploy.yml`: OIDC → `npm ci`
→ lint → typecheck → build → `aws s3 sync` → CloudFront invalidation
of `/index.html` and `/assets/*` only. Required secrets:
`AWS_DEPLOY_ROLE_ARN`, `AWS_REGION`, `S3_BUCKET`, `CF_DIST_ID`.

### CloudFront SPA rewrite (operator setup, out of repo)

Direct navigation to SPA routes (e.g. `/videos/abc`) must reach
`index.html` so the client router can take over, while real files
(`/data/*`, `/assets/*`, legacy `.html`) must pass through to origin
unchanged. AWS's distribution-level **Custom error responses** apply
to all behaviors and cannot be scoped per-prefix, so the standard
"403/404 → /index.html" trick would also clobber missing `/data/*`
JSON requests and break the `NotFoundError` flow.

The supported approach is a **CloudFront Function on
`viewer-request`** that rewrites selectively:

1. Pass through `uri` starting with `/data/` or `/assets/`.
2. Pass through any `uri` whose last segment contains a `.` (covers
   legacy `/{channelId}/{date}_{id}.html` and bare files).
3. Otherwise rewrite `request.uri` to `/index.html`.

Full code and one-time setup steps live in
[docs/superpowers/specs/2026-05-23-vchat-web-spa-design.md](docs/superpowers/specs/2026-05-23-vchat-web-spa-design.md)
§5.4. Provisioning is operator-side and out of scope for this repo.
