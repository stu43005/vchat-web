# vchat-web SPA — Design Spec

Date: 2026-05-23
Topic: A new React SPA (`vchat-web`) that consumes the JSON/JSONL artifacts
produced by honeybee's `chats-archive` component (see honeybee spec
`2026-05-10-chats-archive-jsonl-output-design.md`) and replaces the three
existing server-rendered HTML pages (index, channel, video) with a dynamic
client-rendered UI.

---

## 1. Scope

### In scope

- A React 19 + Vite + TypeScript SPA that renders three pages, all consuming
  JSON/JSONL produced by honeybee under `data/`:
  - **Index page** (`/`) — replaces `index.html`. Two tabs (Live/Upcoming,
    Past) of video cards. Source: `data/index.json`.
  - **Channel page** (`/channels/:channelId`) — replaces
    `{channelId}/index.html`. Channel header + grid of video cards.
    Source: `data/channels/{channelId}.json`.
  - **Video page** (`/videos/:videoId`) — replaces
    `{channelId}/{date}_{videoId}.html`. Video header, aggregates,
    filter chips, virtualized chat list. Source:
    `data/videos/{videoId}.meta.json` + `data/videos/{videoId}.jsonl`.
- Top-bar navigation with logo, breadcrumb, timezone toggle, theme toggle.
- GitHub Actions workflow that builds the SPA and deploys to AWS S3 root
  via OIDC, leaving the honeybee-owned `data/` subtree untouched.
- Legacy fallback: for videos whose `archiveVersion < 2`, the SPA links out
  to the existing static `.html` file in a new tab rather than attempting
  to render the SPA video page.

### Out of scope

- The honeybee JSON/JSONL emission itself (already specified in
  `2026-05-10-chats-archive-jsonl-output-design.md`).
- AWS infrastructure setup (S3 bucket creation, CloudFront distribution,
  OIDC trust policy). The workflow consumes secrets that the user
  provisions once.
- i18n / localization. UI strings are English only.
- Internationalized timestamp formatting beyond timezone selection
  (Local / Asia/Tokyo / UTC).
- Service worker / offline / PWA shell.
- Full-text search across rows.
- Author channel pages (links from chat rows to author YouTube channels
  are not rendered).
- Deep links to a specific chat row within a video.
- Unit tests. Verification is manual (see §10).
- Backfill of historic videos (those with `archiveVersion < 2` keep
  linking to legacy HTML indefinitely; re-archiving is honeybee's
  responsibility).
- Web Worker offload for JSONL parsing. The parser lives behind a single
  function in `src/lib/jsonl.ts` so it can later be swapped for a worker
  without touching call sites; this spec does not implement the worker.

### 4.2 Fetch wrapper (`src/api/client.ts`)

A single helper rooted at `/data/`. Same-origin, no CORS, no auth.

```ts
const DATA_BASE = "/data";

export async function fetchJson<T>(path: string): Promise<T> {
  const res = await fetch(`${DATA_BASE}${path}`, { cache: "default" });
  if (!res.ok) throw new Error(`fetch ${path}: ${res.status}`);
  return res.json() as Promise<T>;
}

export async function fetchJsonl<T>(path: string): Promise<T[]> {
  const res = await fetch(`${DATA_BASE}${path}`, { cache: "default" });
  if (!res.ok) throw new Error(`fetch ${path}: ${res.status}`);
  const text = await res.text();
  return parseJSONL<T>(text);
}
```

`parseJSONL` lives in `src/lib/jsonl.ts`:

```ts
export function parseJSONL<T>(text: string): T[] {
  const out: T[] = [];
  for (const line of text.split("\n")) {
    if (!line) continue;
    out.push(JSON.parse(line) as T);
  }
  return out;
}
```

Isolating the parser behind this function preserves the future option of
swapping in a Web Worker without touching call sites.

### 4.3 Query hooks (`src/api/queries.ts`)

```ts
useIndexQuery(): UseQueryResult<IndexData>
  queryKey: ["index"]
  queryFn: () => fetchJson<IndexData>("/index.json")
  staleTime: 60_000
  gcTime: 5 * 60_000

useChannelQuery(channelId: string): UseQueryResult<ChannelData>
  queryKey: ["channel", channelId]
  queryFn: () => fetchJson<ChannelData>(`/channels/${channelId}.json`)
  staleTime: 60_000
  gcTime: 5 * 60_000

useVideoMetaQuery(videoId: string): UseQueryResult<VideoMeta>
  queryKey: ["videoMeta", videoId]
  queryFn: () => fetchJson<VideoMeta>(`/videos/${videoId}.meta.json`)
  staleTime: 5 * 60_000
  gcTime: 30 * 60_000

useVideoRowsQuery(videoId: string): UseQueryResult<ChatRow[]>
  queryKey: ["videoRows", videoId]
  queryFn: () => fetchJsonl<ChatRow>(`/videos/${videoId}.jsonl`)
  staleTime: 5 * 60_000
  gcTime: 30 * 60_000
```

Components never call `fetch` directly. The `meta` query fires first on
the video page; `rows` fires in parallel (no dependency between them).

## 5. Routing

### 5.1 Route table

| Path | File | Loader/Search params |
| --- | --- | --- |
| `/` | `routes/index.tsx` | search: `{ tab: "live" \| "past" }` default `"live"` |
| `/channels/$channelId` | `routes/channels.$channelId.tsx` | param: `channelId` |
| `/videos/$videoId` | `routes/videos.$videoId.tsx` | param: `videoId`; search: `{ types?: string[], minSig?: number }` |

Search-param schemas use `zod` and are passed to TanStack Router's
`validateSearch`. Unknown search params are dropped silently. Default
values are applied when params are absent.

### 5.2 Search-param semantics on video page

- `types` — comma-separated list of row types to show. Valid values:
  `chat`, `superChat`, `superSticker`, `membership`, `membershipGift`,
  `membershipGiftPurchase`, `milestone`, `poll`, `raid`. (`raidOutgoing`
  is grouped under `raid` — selecting `raid` shows both.) Default
  (when param absent): `["superChat", "superSticker"]`.
- `minSig` — integer 0..7. Only filters `superChat` and `superSticker`
  rows; other types are unaffected. A row with no `significance` field
  is treated as significance `0`. Default: `1`.

Filter changes update the URL via `navigate({ search: ..., replace: true })`
so back/forward history is not polluted on every chip toggle.

### 5.3 BrowserRouter on S3

CloudFront must rewrite 404s on non-asset paths to `/index.html`. This is
a one-time CloudFront configuration done outside this repo. The Vite
build emits hashed assets under `/assets/*` so the rewrite rule should
exclude `/assets/*` and `/data/*` (both serve real files).

## 6. Page-by-page UX

### 6.1 TopBar (rendered on all pages)

Single `AppBar position="sticky"` with `Toolbar`:

- Left: `VChat` title (text logo), clickable → `/`.
- Middle (grows): MUI `Breadcrumbs`. Derived from
  `useMatches()` (TanStack Router):
  - `/` → `Home`
  - `/channels/:id` → `Home / {channel.name}` (channel name from
    `useChannelQuery`; placeholder `…` while loading)
  - `/videos/:id` → `Home / {channel.name} / {video.title}` (truncated
    with ellipsis; channel name from `useVideoMetaQuery().channel.name`)
- Right: timezone select (`IconButton` opening a `Menu` with three
  options: `Local`, `Asia/Tokyo`, `UTC`) + theme select (`IconButton`
  opening a `Menu` with three options: `Light`, `Dark`, `System`).
  Both selections persist in `localStorage` (keys `vchat:timezone`,
  `vchat:theme`).

Mobile: breadcrumb collapses to last crumb only when viewport <
`sm` breakpoint.

### 6.2 Index page (`/`)

Layout:

1. `Tabs` with values `live` and `past`, controlled by URL search param
   `tab`. Labels: `Live / Upcoming ({live.length})`,
   `Past ({past.length})`.
2. `Grid container spacing={2}` with `VideoCard` children
   (`Grid xs={12} sm={6} md={4} lg={3}`).

States:

- Loading: 8 `Skeleton` cards.
- Error: `Alert severity="error"` with a retry `Button` calling
  `refetch()`.
- Empty tab: centered `Typography color="text.secondary"`:
  `"No live streams"` or `"No past archives"`.

### 6.3 Channel page (`/channels/:channelId`)

Layout:

1. `ChannelHeader` (avatar 64px circular + channel name `Typography variant="h4"`
   + external link icon button → `https://www.youtube.com/channel/{channelId}`).
2. Same responsive `Grid` of `VideoCard`s. On this page each card omits
   the channel name/avatar (redundant; controlled by a `hideChannel`
   prop on `VideoCard`).

States: same loading/error/empty patterns as Index. 404 (file not found)
shows `"Channel not found"` empty state.

### 6.4 Video page (`/videos/:videoId`)

Layout (vertical stack inside `Container maxWidth="lg"`):

1. **VideoHeader**
   - Thumbnail (left, 240px wide, 16:9), clickable →
     `https://youtu.be/{videoId}` (new tab).
   - Title (`Typography variant="h5"`, clickable → same URL).
   - Channel chip (clickable → `/channels/:channelId`).
   - Status line: same logic as honeybee's `StatusText` but using
     `meta.status` + `scheduledStart` / `availableAt`:
     - `upcoming` → `"Start at {scheduledStart}"` (or `"Upcoming"` if
       no `scheduledStart`)
     - `live` → `"Live Now"` (red)
     - `past` / `missing` → `"Published at {availableAt}"`
2. **CurrencyTable** — from `meta.aggregates.currencyTable` + `jpyTotal`.
   Compact MUI `Table`, monospace via `sx={{ fontFamily: "monospace" }}`,
   right-aligned numeric cells. Columns: `symbol`, `code`, `sum`,
   `sum (JPY)`. Final row: blank symbol/code/sum, JPY total.
3. **AggregatesSummary** — single row of `Chip`s showing counts from
   `meta.aggregates`: Chat, SC, Sticker, Member, Gift (one chip per of
   gift count + total gift amount), Milestone, Poll, Raid. Format
   numbers with `toLocaleString()`.
4. **FilterChips**
   - Row of toggleable MUI `Chip`s, one per type listed in §5.2 (9
     chips; `raid` controls both `raid` and `raidOutgoing`). Each chip
     shows the type name + total count from `meta.aggregates.*Count`
     in parentheses. Selected state styled with `color="primary"`.
   - Below the chip row: a single MUI `Slider` (`min=0`, `max=7`,
     `step=1`, marks at every integer) labeled
     `"Min significance (SuperChat / Sticker)"`.
5. **ChatList** — see §7.

States:

- `meta` loading or `rows` loading: full-page `Skeleton` block (header,
  table, chips, ten row-skeletons).
- `meta` error: `Alert severity="error"` + retry.
- `rows` error: render header + meta as normal; show error `Alert` in
  place of the chat list with retry.
- `meta.archiveVersion < 2` (defensive — normal navigation cannot reach
  here because cards redirect): full-page `Alert severity="warning"`
  with a button "Open legacy archive" linking to the legacy HTML URL
  (§8). The chat list is not rendered.

### 6.5 Legacy fallback on `VideoCard`

`VideoCard` checks `video.archiveVersion`:

- If `archiveVersion >= 2`: card's title and thumbnail navigate via
  TanStack Router `<Link to="/videos/$videoId" params={{ videoId: id }} />`.
- If `archiveVersion < 2`: same elements become plain
  `<a target="_blank" rel="noopener" href={legacyVideoHref(video)}>`
  where `legacyVideoHref` computes
  `/${video.channel.id}/${YYYYMMDD(video.availableAt, "Asia/Tokyo")}_${video.id}.html`
  (mirroring honeybee's `getVideoPath`). A small MUI `Chip label="Legacy"
  size="small"` is overlaid in the top-left of the thumbnail.

The channel name link always uses TanStack Router; archive version does
not affect channel-page reachability.

## 7. Chat list rendering

### 7.1 Filtering

Filtering is pure-client. The full row array from `useVideoRowsQuery` is
passed through a `useMemo` filter:

```ts
const filtered = useMemo(() => {
  const typeSet = new Set(selectedTypes);
  const raidGroup = typeSet.has("raid");
  return rows.filter((row) => {
    if (row.type === "raidOutgoing") return raidGroup;
    if (!typeSet.has(row.type)) return false;
    if (row.type === "superChat" || row.type === "superSticker") {
      return (row.significance ?? 0) >= minSig;
    }
    return true;
  });
}, [rows, selectedTypes, minSig]);
```

Filter operations on a 50k-row array run in ~5ms on commodity hardware;
no further optimization in this spec.

### 7.2 Virtualization

`@tanstack/react-virtual` `useVirtualizer` over the `filtered` array.
Configuration:

- `count: filtered.length`
- `getScrollElement: () => scrollRef.current`
- `estimateSize: () => 64` (rows are variable height; this is the
  initial estimate; the virtualizer measures actual heights and
  re-estimates)
- `overscan: 8`

The scroll container is a `Box` with `height: calc(100vh - 320px)` (top
bar + header + chips occupy ~320px above; tweak in implementation).
Each virtualized row is wrapped in a `<div ref={virtualRow.measureRef}>`
so dynamic heights are measured.

### 7.3 Row components

Ten components under `src/components/chat-rows/`, dispatched by `type`.
Each receives `{ row, video, no, timezone }` and renders consistent
columns:

| Column | Width | Content |
| --- | --- | --- |
| No. | 56px right-aligned | running index within `filtered` |
| Timestamp | 180px | YouTube-time link or plain text (§7.4) |
| Author photo | 56px | `Avatar src={authorPhoto}` (skipped for poll/raid types) |
| Author | 200px | author name (skipped for poll/raid types) |
| Body | flex | type-specific (see below) |

Type-specific body:

| Type | Body |
| --- | --- |
| `chat` | `row.message` |
| `superChat` | currency badge (left, `bgcolor={row.color}`) + amount (with `formatCurrency`, plus JPY conversion when `currency !== "JPY"`) + message (italic "(wordless superchat)" placeholder when `message` is null) |
| `superSticker` | currency badge + amount + `<img src={row.image} title={row.text}>` |
| `membership` | `"Joined as a member ({row.level ?? row.membership ?? 'N/A'})"`, green chip on left |
| `membershipGift` | `"Received a membership gift from {row.senderName ?? 'N/A'}"`, green chip |
| `membershipGiftPurchase` | `"Purchased {row.amount} membership gift(s)"`, green chip |
| `milestone` | `row.message ?? "(wordless milestone)"`, green chip |
| `poll` | "Poll" in Author column (photo skipped); body: `"{voteCount} votes"` line (if present) + question + bulleted choices each with `(xx.x%)` when `voteRatio` is present |
| `raid` | "Raid (incoming)" in Author column; body: `"{row.sourceName} and their viewers just joined. Say hello!"` |
| `raidOutgoing` | "Raid (outgoing)" in Author column; body: `"Sending you to {row.originName ?? ''}"` |

`formatCurrency` matches honeybee's helper: `Intl.NumberFormat("en-US")`
(was `ja-JP`; switched to `en-US` to match the English-only UI policy)
with `style: "currency"`, `currency: row.currency`, fraction digits
derived from a small currency table copied/extracted from honeybee
(`src/lib/format.ts` keeps a static `currencyDigits` map). When the
currency code is unknown in the map, fall back to 2 fraction digits.

### 7.4 Timestamp display

`formatTimestamp(iso, timezone)` returns `YYYY-MM-DD HH:mm:ss` in the
selected zone. Built using `Intl.DateTimeFormat` with `timeZone` parameter:

- `Local` → omit `timeZone` (uses browser default)
- `Asia/Tokyo` → `timeZone: "Asia/Tokyo"`
- `UTC` → `timeZone: "UTC"`

If `meta.actualStart` is defined and `row.timestamp >= meta.actualStart`,
the timestamp wraps in `<a target="_blank" rel="noopener"
href={getYouTubeUrl(videoId, secondsFromStart)}>` where
`secondsFromStart = floor((Date.parse(timestamp) - Date.parse(actualStart)) / 1000)`
and `getYouTubeUrl(id, t)` returns `https://youtu.be/${id}?t=${t}`.
Otherwise the timestamp renders as plain text (no link).


   - `role-to-assume: ${{ secrets.AWS_DEPLOY_ROLE_ARN }}`
   - `aws-region: ${{ secrets.AWS_REGION }}`
8. `aws s3 sync dist/ s3://${{ secrets.S3_BUCKET }}/ --delete --exclude "data/*"`
   — `--delete` removes stale SPA assets; `--exclude "data/*"` ensures
   the honeybee-owned subtree is never touched even if a future build
   accidentally emits files under `data/`.
9. `aws cloudfront create-invalidation
   --distribution-id ${{ secrets.CF_DIST_ID }}
   --paths "/index.html" "/assets/*"`
   — invalidates only SPA paths; `/data/*` is never invalidated here
   (honeybee's sync job is responsible for its own cache busting).

Required GitHub secrets (provisioned by user out-of-band):
`AWS_DEPLOY_ROLE_ARN`, `AWS_REGION`, `S3_BUCKET`, `CF_DIST_ID`.

If any of `lint`, `typecheck`, `build` fails, the deploy step does not
run (GitHub Actions default behavior on step failure).

## 10. Verification (manual)

After deployment to a non-production S3 bucket pointed at a real
`data/` tree, perform the following checks:

1. Load `/`. Confirm Live/Upcoming and Past tabs render the same set
   of cards (same count, same order) as the legacy `/index.html`.
2. Toggle the Tabs; URL `?tab=` updates; browser back/forward navigates
   between tabs.
3. Click a card with `archiveVersion: 2`. Confirm SPA navigates to
   `/videos/:id` and the video page renders.
4. Click a card with `archiveVersion: 1` (legacy). Confirm a new tab
   opens to the legacy HTML at
   `/{channelId}/{YYYYMMDD}_{videoId}.html`. The current SPA tab does
   not navigate. The card shows a "Legacy" chip.
5. On a video page, confirm `CurrencyTable` totals match the legacy
   HTML's currency table exactly.
6. Confirm `AggregatesSummary` chip counts match the row counts a user
   would see by enabling all filters in the legacy HTML.
7. Toggle filter chips; URL `?types=` updates; row list filters
   correctly; running No. column renumbers from 1.
8. Move the significance slider from 1 to 7; SuperChat/Sticker rows
   below the threshold disappear; other types are unaffected.
9. Click a timestamp on a row where `timestamp >= actualStart`. A new
   tab opens to `https://youtu.be/{videoId}?t={seconds}` where
   `seconds` equals `floor((row.timestamp - actualStart) / 1000)`.
10. For a row where `actualStart` is missing or `timestamp < actualStart`,
    the timestamp is plain text (not a link).
11. Switch timezone in TopBar from `Local` → `Asia/Tokyo` → `UTC`.
    Confirm all visible timestamps (header status, currency table N/A,
    chat rows) reformat consistently. Setting persists across reload.
12. Switch theme `Light` → `Dark` → `System`. Setting persists across
    reload. `System` follows OS preference.
13. Navigate to `/channels/:channelId`. Confirm header (avatar + name +
    YouTube link icon) and video grid match the legacy
    `{channelId}/index.html`. Cards omit channel name (since it's in
    the header).
14. Scroll a video page with >10k rows. Confirm:
    - The scroll container is smooth (no full re-render on each scroll).
    - DOM size stays bounded (a few dozen row nodes, not 10k).
    - The browser tab does not OOM.
15. Visit `/videos/:nonExistentId`. Confirm an error `Alert` is shown
    with a retry button (404 from `fetchJson` becomes a thrown error
    surfaced by React Query).
16. Direct-load a deep URL (e.g.
    `https://{host}/videos/abc?types=superChat&minSig=3`) without
    going through `/` first. Confirm CloudFront serves `index.html`
    and the SPA bootstraps directly into the video page with the
    correct filter state.
17. After a deploy, confirm CloudFront invalidation includes
    `/index.html` and `/assets/*` but not `/data/*`. The legacy HTML
    files under `/{channelId}/...` are unaffected.

## 11. Constraints on implementation

- No source-code comments referencing this design document — no section
  markers, no "spec"/"plan" mentions, no "Task N". Field names and
  identifiers are expected to be self-explanatory.
- No unit tests in this PR. Verification is manual per §10.
- No silent fallbacks on schema mismatch: if a JSONL row's `type` is
  unknown, drop it from the filtered list (do not crash) but log a
  `console.warn(\`unknown row type: \${type}\`)` once per session per
  type. This is the only "telemetry" the SPA emits.
- No analytics, no error reporting service, no third-party scripts.
- All external links open in a new tab with `rel="noopener noreferrer"`.
- Avatars (`authorPhoto`, channel `avatarUrl`) load with
  `loading="lazy"` and `referrerPolicy="no-referrer"` to keep YouTube
  thumbnail hot-link behavior consistent.
